Presentation

The initial idea of the project was to redesign the intranet page of our school, which was quite old. Once the idea was launched, one of the school’s supervisors told us one of his ideas, which was to create an interface that all students could consult in order to know the times of passage of the canteen (which was and is always a chaotic organization).

So it was from this idea to create a utilitarian website for our school that we left without any experience in web development ...

Realisation

First, we had to realize that this project would require skills that we did not yet possess. So we started from the CSS of HTML5 which was the one already used by the school before ; maked of more than 5000 lines, the time to understand how everything worked was quite long ...
Then we decided to use the MVC architecture to organize the site as well as possible.

Then, the hardest part was that of PHP, MySQL, and database management, which had to include all classes, with their times of passage, depending on the days and the week ... A real puzzle ...
Without going into detail, the data table consisted only of numbers that referred to data from other tables.

For the design of the page, we went to simple search bars, as there was before because it is the page that appears as soon as a tab is opened on one of the computers of the school. In addition, a list of useful links for students and teachers had to be introduced. For this, semi-transparent colour blocks were used, containing the links of what they indicated. We added to this a background image with logos for each block.

Result

The result was quite satisfactory, and liked by most of the students, as well as the administration, before which the project had to be presented ... several times ... Regarding the passage of the canteen, the interface is minimalist: a button that leads to another page, on which we can see the passage, updated live by a supervisor.

The website is still online and currently un use.
